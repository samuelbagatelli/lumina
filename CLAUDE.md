## Architecture

**Lumina** is a book tracking PWA built with:
- **Stack**: React 19, TypeScript, Vite, Tailwind CSS 4
- **Backend**: Firebase (Auth + Firestore)
- **Routing**: React Router v7

### Project Structure

```
src/
├── components/     # Reusable UI components
│   ├── books/     # Book-related: AddBookForm, BookCard, CollectionManager
│   └── layout/    # Layout, Navbar
├── pages/         # Route pages: Home, AddPage, Explore, BookDetail
├── contexts/      # React contexts: AuthProvider, ThemeProvider
├── hooks/         # Custom hooks: useBooks, useCollections
├── lib/           # Firebase setup, utility functions
└── types.ts       # TypeScript types for Book, ArchiveCollection
```

### Data Model

- **Book**: Contains title, author, pages, status (Want to Read/Reading/Completed), progress (0-100%), genre, coverUrl, timestamps
- **ArchiveCollection**: Hierarchical collections (name, parentId, userId)
- User-scoped data stored at `users/{userId}/books` and `users/{userId}/collections`

### Key Patterns

1. **Auth Context**: Firebase Google sign-in, user state managed via `onAuthStateChanged`
2. **Theme Context**: Light/dark mode with localStorage persistence
3. **useBooks Hook**: Manages CRUD operations with Firestore, automatically syncs via `onSnapshot`
4. **Firestore Security Rules**: Strict validation on book/collection operations, owner-only access
5. **Path convention**: All Firestore operations use `users/${user.uid}/books` or `users/${user.uid}/collections`

### Configuration

- `vite.config.ts`: Resolves `@/*` to `./src`, injects `GEMINI_API_KEY` for AI Studio
- `firebase-applet-config.json`: Contains Firebase config (auto-generated by AI Studio)
- `firestore.rules`: Security rules enforcing user-scoped access
